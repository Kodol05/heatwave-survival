# 게임 제작 흐름

각 단계가 끝나면 **실제로 동작하는 버전**이 나오게 설계되어 있어요.  
다음 단계 시작 전에 현재 단계가 동작하는지 꼭 확인하세요.

---

## Phase 0: 환경 셋업 (1단계)

### 목표
브라우저에 빈 Phaser 화면이 뜬다.

### 작업
1. 프로젝트 폴더 생성
2. `index.html` 생성 (Phaser 3 CDN 로드)
3. `src/main.js` 생성 (Phaser 게임 인스턴스 초기화)
4. 로컬 서버 실행 (예: VS Code Live Server, `python -m http.server`)
5. 빈 검은 화면 또는 색깔 배경이 보이는지 확인

### 완료 기준
- [ ] 브라우저에서 Phaser 캔버스가 보임
- [ ] 콘솔에 에러 없음

---

## Phase 1: 씬 골격 만들기 (2단계)

### 목표
씬 5개가 골격만 존재하고, 서로 전환된다.

### 작업
1. 씬 5개 빈 클래스 생성
   - `BootScene` (에셋 로드용, 일단 비워둠)
   - `HomeScene` (집)
   - `TimeSelectScene` (시간 선택)
   - `ErrandScene` (심부름)
   - `FirstAidScene` (응급처치)
   - `ResultScene` (결과)
2. 각 씬에 임시 텍스트 표시 (예: "여기는 집 화면입니다")
3. 각 씬에 임시 버튼 또는 스페이스바로 다음 씬으로 전환
4. 씬 전환 흐름 테스트

### 완료 기준
- [ ] 모든 씬이 서로 연결됨
- [ ] 키보드/클릭으로 씬 전환 확인

---

## Phase 2: 허브 (집) 만들기 (3단계)

### 목표
집 화면에서 어머니와 컴퓨터에 상호작용할 수 있다.

### 작업
1. `HomeScene`에 배경 placeholder 추가 (단색 사각형)
2. 어머니 NPC placeholder 추가 (색깔 사각형)
3. 컴퓨터 placeholder 추가 (다른 색 사각형)
4. 플레이어 캐릭터 placeholder 추가 + 이동 (방향키 또는 WASD)
5. 어머니/컴퓨터에 가까이 가면 "상호작용 가능" 표시
6. 스페이스 또는 E 키로 상호작용
   - 어머니 → `TimeSelectScene`로 이동
   - 컴퓨터 → `FirstAidScene`로 이동

### 완료 기준
- [ ] 플레이어가 집 안에서 자유롭게 움직임
- [ ] 어머니/컴퓨터 상호작용으로 각 씬에 진입됨

---

## Phase 3: 시간 선택 화면 (4단계)

### 목표
8~20시 중 하나를 선택하면 그 시간 정보를 들고 심부름 씬으로 넘어간다.

### 작업
1. `TimeSelectScene`에 시간 버튼 13개 (8시~20시) 배치
2. 각 버튼에 시간/온도/난이도 표시
3. 마우스 오버 시 설명 텍스트 표시
4. 클릭 시 선택한 온도 값을 `ErrandScene`로 전달

### 완료 기준
- [ ] 13개 시간 버튼이 보임
- [ ] 호버 시 설명 표시
- [ ] 선택한 시간이 다음 씬에 전달됨 (콘솔로 확인)

---

## Phase 4: 심부름 게임 - 기본 (5단계)

### 목표
캐릭터가 자동 전진하고, 상하 이동이 가능하며, 결승점에 도달하면 클리어된다.

### 작업
1. `ErrandScene` 배경 (스크롤 placeholder, 단색 + 줄무늬 정도)
2. 플레이어 캐릭터 placeholder
3. 자동 전진 로직 (카메라가 따라감 또는 배경이 흐름)
4. 상하 이동 (방향키/WASD)
5. 화면 경계 처리 (위/아래로 못 나가게)
6. 일정 거리 도달 시 클리어 → `ResultScene`로 이동

### 완료 기준
- [ ] 캐릭터가 자동 전진
- [ ] 상하 이동이 자연스러움
- [ ] 결승점 도착 시 결과 화면

---

## Phase 5: 게이지 시스템 (6단계)

### 목표
더위/수분 게이지가 화면에 표시되고, 시간이 지나면서 변화한다.

### 작업
1. 더위 게이지 UI (상단 또는 좌측)
2. 수분 게이지 UI (그 옆)
3. 수분이 시간당 자동 감소
4. 더위가 일정 이상이면 수분 감소 속도 가속
5. 게이지가 위험 수치 도달 시 게임 오버 → `ResultScene`

### 완료 기준
- [ ] 두 게이지가 화면에 보임
- [ ] 가만히 있어도 수분이 줄어듦
- [ ] 게임 오버 트리거 동작

---

## Phase 6: 지형 시스템 (7단계)

### 목표
그늘/일반/아스팔트 위에 있느냐에 따라 더위가 다르게 변한다.

### 작업
1. 지형 placeholder 3종 (색깔로 구분, 그늘=어두운 회색, 일반=베이지, 아스팔트=검정)
2. 지형 배치 데이터 (간단한 배열 또는 패턴)
3. 플레이어 위치 기준 현재 지형 판정
4. 지형별 더위 변화 적용
5. 시간대(온도)에 따라 일반/아스팔트의 더위 상승 속도 가변

### 완료 기준
- [ ] 3종 지형이 시각적으로 구분됨
- [ ] 그늘 위에선 더위가 회복됨
- [ ] 아스팔트 위에선 더위가 빨리 오름
- [ ] 14시 선택 시 8시보다 확실히 어려움

---

## Phase 7: 아이템 시스템 (8단계)

### 목표
7종 아이템이 길에 나타나고, 먹으면 게이지가 변한다.

### 작업
1. 아이템 데이터 정의 (이름, 수분 효과, 더위 효과)
2. 아이템 placeholder 7종 (색깔/글자로 구분)
3. 아이템 스폰 로직 (랜덤 배치 또는 패턴)
4. 충돌 감지 → 효과 적용
5. 획득 시 시각 피드백 (텍스트 팝업 등)

### 완료 기준
- [ ] 7종 아이템이 모두 등장함
- [ ] 물은 좋고, 핫초코/라면은 더위가 올라감을 확인
- [ ] 획득 시 피드백 표시됨

---

## Phase 8: 결과 화면 (9단계)

### 목표
심부름 끝난 후 점수와 획득 아이템 결과가 표시되고, 집으로 돌아간다.

### 작업
1. `ResultScene`에 클리어/실패 분기
2. 점수 계산 표시
3. 획득한 아이템 목록 표시
4. "집으로 돌아가기" 버튼 → `HomeScene`

### 완료 기준
- [ ] 클리어/실패 메시지가 다름
- [ ] 점수와 아이템 목록이 정확함
- [ ] 집으로 돌아가서 다시 시작 가능

---

## Phase 9: 응급처치 게임 (10단계)

### 목표
컴퓨터로 응급처치 미니게임을 5문제 풀 수 있다.

### 작업
1. 5개 상황 데이터 정의 (상황, 선택지 4개, 정답)
2. 상황 표시 UI
3. 선택지 버튼 4개
4. 타이머 바 UI (줄어드는 애니메이션)
5. 판정 로직 (Perfect/Good/Miss/오답)
6. 5문제 연속 진행
7. 총점 표시 → `HomeScene`로 복귀

### 완료 기준
- [ ] 5문제가 순서대로 등장
- [ ] 타이머 바가 정확히 동작
- [ ] 판정이 빠르기에 따라 다르게 나옴
- [ ] 최종 점수 표시

---

## Phase 10: 다듬기 (11단계 이후, 선택)

### 목표
완성도 높이기.

### 작업
- 사운드/BGM 추가
- 애니메이션 다듬기
- 수치 밸런싱
- placeholder 그래픽을 실제 그래픽으로 교체 (이건 언제든 가능하게 구조 잡혀 있음)
- 3번째 미니게임 추가 (TV)

---

## 작업 순서 요약

```
0. 환경 셋업
1. 씬 골격
2. 허브 (집)
3. 시간 선택
4. 심부름 기본 (이동/전진/결승점)
5. 게이지 시스템
6. 지형 시스템
7. 아이템 시스템
8. 결과 화면
9. 응급처치 게임
10. 다듬기 (선택)
```

각 Phase는 **독립적으로 동작 확인 후** 다음으로 넘어가는 게 중요해요.  
중간에 막히면 그 Phase 안에서만 디버깅하면 되니까 추적이 쉬워요.
